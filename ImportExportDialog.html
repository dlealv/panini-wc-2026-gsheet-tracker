Export current sticker counts from the COUNTRIES and COUNTS named ranges in the Stickers tab. Load a text/CSV file or paste its content. Data will be written to the COUNTS named range in the Stickers tab, preserving formatting.
1) Upload file
When a file is selected, its content will be loaded into the text area below.
2) Or paste content
3) Loading mode
  • Import data: validates country codes against the COUNTRIES named range, clears all values in the COUNTS named range, then loads the input data.
  • Update counts clearing country counts: clears only the rows for countries present in the input, then loads those countries again.
  • Update counts: only overwrites sticker positions explicitly provided in the input; all other values remain unchanged.
Input rules
  • Format: [flag] CODE,number[,number(repeats)][,start-end][,start-end(repeats)]...
  • Use comma , as delimiter.
  • Single sticker values are supported. Repeats in parentheses, sticker ranges, and a leading flag icon are optional.
  • Example: 🇲🇽 MEX,1,2,3(2),5-8,10-12(2)
    • Flag 🇲🇽 before the country code is optional and ignored by the parser.
    • Range 5-8 is interpreted as stickers 5,6,7,8.
    • Range 10-12(2) is interpreted as stickers 10(2),11(2),12(2).
  • Country code must exist in the COUNTRIES named range.
  • Sticker 0 only exists for FWC. For other countries it is accepted and mapped to count 0.
  • Sticker 20 only exists for non-FWC countries. For FWC it is accepted and mapped to count 0.
Actions
  • Validate / Preview: checks syntax and shows how values will be mapped without writing to the sheet.
  • Clear: removes the selected file, clears the pasted input, clears the preview, and resets the message.
  • Import: writes the validated values to the sheet using the selected loading mode.
  • Cancel: closes the dialog without importing anything.
Export result
The exported content uses the same syntax accepted by the import process. Only sticker counts greater than 0 are included.
Actions
  • Download file: downloads the exported sticker data as a local file, usually into your browser's default Downloads folder.
  • Copy: copies the exported text to the clipboard.
  • Close: closes the dialog.
